# 资讯聚合爬虫改造与最终验证日志

## 1. 改造目标

本次改造针对原程序只能读取静态详情页、把列表页标题（例如“综合新闻”）误当正文、不能稳定翻页、重复抓取、动态页面无法取数以及数据库可能写入乱码/HTML 壳等问题。

最终目标如下：

1. 从每个配置种子的列表页筛选真实详情链接，并进入子链接提取标题、日期、正文和主图。
2. 从第一页检查最新内容，再按持久化游标继续历史分页。
3. 每个种子每次运行最多新增 20 条；只有 SQLite 真正插入一条新 URL 才消耗一个配额。
4. 已入库 URL 不重复下载正文，不重复入库；失败、乱码、栏目壳、二进制内容不计配额。
5. 纯图片通知把正文主图 URL 写入 `news.image`，不把图片二进制写入正文。
6. 数据库允许其他程序并发读取和短写入，不被爬虫长期锁住。

## 2. 原始问题、定位与修改

| 问题位置 | 现象和根因 | 修改 | 效果 |
|---|---|---|---|
| 通用列表页 | 只得到“综合新闻”等栏目名；原逻辑没有可靠地区分列表、导航和详情链接 | 在 `crawler_rules.cpp` 增加 URL 标准化、相对地址解析、同域限制、资源后缀排除、详情特征评分和导航负分 | 列表壳不再作为新闻；只把高可信详情交给正文解析 |
| 图书馆 Engine2 | 列表由接口动态返回，静态 HTML 中没有完整资讯 | 解析入口页的 `appId`、`engineInstanceId`、`sign`，调用 `/engine2/general/.../more-datas` POST 接口，生成 `/engine2/d/...` 详情 URL | 已验证第 1～3 页连续取数，正文和标题来自具体详情，不再只得到“综合新闻” |
| 普通分页 | 只处理第一页，下一次运行重复第一页 | `HTMLparsing (1).cpp` 增加下一页识别；`database.h/.cpp` 增加 `crawl_progress`，保存普通列表下一页 URL 和 Engine2 页码 | 每次先检查第 1 页新资讯，再跳到历史游标继续回填 |
| 配额 | 请求次数被当成成功次数，垃圾链接会浪费种子配额 | `main.cpp` 将配额固定为每种子 20 条，仅 `database::addNews()` 确认实际插入后计数；另设 80 次详情安全上限 | 重复 URL、失败链接、乱码和无正文页面均不消耗 20 条新增配额 |
| 跨页重复导航 | 人事处等站点每页重复出现相同师资/栏目链接 | 去重集合从“单页”提升为“整个种子本轮” | 同一垃圾链接一轮最多尝试一次，不随翻页反复请求 |
| 增量去重 | 每次重新下载已入库正文 | `news.url` 唯一约束 + 下载前 `newsExists()` | 已知 URL 直接跳过；匹兹堡学院复跑验证数据库 `481 → 481`，新增 0 |
| 就业中心 | 页面大且结构畸形，过期详情会跳回门户首页 | 按 `news-content-tit` 和 `mycontent` 直解析；详情 URL 回到门户时直接拒绝 | 能提取真实招聘/通知；门户壳不再入库，也不再卡住 Tidy |
| 国际处 | 一篇约 319 KB 的深层嵌套 HTML 让 Tidy 持续占用 CPU | 按 `news-head`、`news-content` 直解析，并限制处理片段大小 | 定点测试 25.6 秒跨 2 页新增 20 条；不再单篇卡死数分钟 |
| 华西口腔 | 每篇约 90 KB，通用解析约 12 秒；旧实验栏目还会混入静态页 | 按 `article_title`、`article_cont` 直解析；增加静态标题门槛 | 单站 20 个候选约 14 秒完成；“帮助信息”“版权声明”等被拒绝 |
| 工会 | Office 标签和 3.37 MB 异常详情可触发 libtidy 访问冲突 | 按 `info-box`、`content-box` 直解析；不符合模板的 `showProNews` 详情直接失败，不进入 Tidy | 定点测试新增 20/20，已知 15，失败 2，正常退出 |
| 生命学院 | 列表里的非详情栏目页进入 Tidy 后崩溃 | 真实通知限定 `/info/...`；非详情栏目壳禁止进入 Tidy | 16.5 秒跨 3 页新增 20 条，进程正常退出 |
| 哲学系 | 全站导航中的教师、公众号页面被详情评分选中 | 该通知栏目只接受 `/info/1016/...` | “短期讲座教师”“微信公众号矩阵”等静态页已删除并被规则阻断 |
| HTML/乱码污染 | VSB 边界落在 `id="div_vote_id"` 属性中间，留下 `<div`；Engine2 富文本解析失败时保存原 HTML | 边界回退到完整标签起点；Engine2 机械去除脚本、样式和标签；数据库增加 UTF-8/HTML 二次质量门槛 | 全库 HTML 残片和非法 UTF-8 均为 0 |
| 中文截断 | `substr(0, 8000)` 可能从 UTF-8 汉字中间截断，制造非法尾字节 | 增加 `utf8_prefix()`，正文 8000 字节和摘要 240 字节都按 UTF-8 字符边界截断 | 人事处定点复测新增 20/20，长中文正文可正常入库 |
| 图片通知 | 只有图片而正文很短时被误判失败，或把图标当主图 | 提取 `src/data-src/data-original`，排除 logo、二维码、附件类型图标；图片型 JSON 也可写入 `news.image` | 当前数据库有 392 条有效 HTTP(S) 图片 URL，无图标类污染 |
| SQLite 并发 | 担心爬虫长期锁住动态数据库 | 启用 WAL、`synchronous=NORMAL`、5 秒 busy timeout；网络请求期间不持有事务，所有语句及时 finalize，退出时 close | 爬虫运行时外部连接可同时读取，并成功取得写事务后回滚 |

## 3. 主要代码文件

- `main.cpp`
  - 每种子 20 条新增配额。
  - 80 次详情请求安全上限。
  - 第 1 页查新、历史游标续爬、整种子去重。
  - 输出每个种子的 `inserted=x/20`、known、failed 和是否满额。
  - 标题、正文、UTF-8、HTML 壳和图片通知质量检查。
- `crawler_rules.h/.cpp`
  - URL 解析、标准化、相对地址合并、同域判断。
  - 详情链接评分、垃圾链接和资源文件过滤。
  - 哲学系等站点的栏目路径约束。
- `HTMLparsing.h`、`HTMLparsing (1).cpp`
  - 普通列表、Engine2 接口和下一页提取。
  - VSB、就业中心、国际处、华西口腔、工会等模板直解析。
  - JSON/动态数据提取、图片提取、HTML 清洗和 UTF-8 安全截断。
- `database.h/.cpp`
  - `news` URL 唯一约束。
  - `crawl_progress` 分页游标。
  - `crawl_failures` 失败次数与错误记录。
  - WAL 并发、busy timeout、入库前二次质量检查。
- `CampusNewsSystem.vcxproj`
  - Debug/Release x64 均启用 C++17、`/utf-8` 及必要宏。
- `config.json`
  - 材料学院失效地址改为当前公告栏目 `https://mse.scu.edu.cn/xygk/xygg.htm`。

## 6. 后续运行逻辑

1. 程序启动后打开 WAL 数据库，不开启跨网络请求的长事务。
2. 每个种子先抓第 1 页，发现新 URL 才下载正文并尝试入库。
3. 已知 URL 直接跳过，不消耗配额。
4. 新 URL 必须通过正文/图片/UTF-8/HTML 质量门槛，SQLite 实际插入成功后才将配额加 1。
5. 第 1 页处理后，从 `crawl_progress` 继续历史分页，直到新增 20 条、列表结束或达到 80 次详情安全上限。
6. 程序退出时 finalize 所有语句并关闭数据库连接。

正常运行命令：

```powershell
.\bin\x64\Release\CampusNewsSystem.exe config.json
```

只扫描并验证、不写新闻数据：

```powershell
.\bin\x64\Release\CampusNewsSystem.exe config.json --scan-only
```

## 7. 已知外部限制

当前 7 个 WAF 站点返回的是验证/拦截响应，不是真实列表 HTML。已尝试普通 HTTP、压缩、HTTP/1.1 和无头浏览器方式，当前环境仍无法稳定通过。程序会明确记录 HTTP 202/412，不会把验证页、JavaScript 壳或乱码写入数据库，也不会把这些失败计入 20 条新增配额。

## 8. 运行时崩溃防护补充（2026-07-15）

- 现象：Debug 程序访问材料学院 `https://mse.scu.edu.cn/ggl.htm` 后异常退出。
- 定位：Windows 应用程序事件记录显示 `tidyd.dll` 抛出 `0xc0000005` 访问冲突；数据库完整性检查正常。
- 原因：材料学院的栏目壳被当成详情候选，畸形 HTML 进入 libtidy 原生修复逻辑。C++ `try/catch` 无法捕获这种 Windows 原生访问冲突。
- 修改：材料学院和生命科学学院的非 `/info/` 页面只提取候选链接，不进入正文 Tidy 回退；Tidy 调用和清理调用增加 MSVC SEH 防护，单页触发原生异常时记录警告、放弃该页并继续其余任务。
- 构建验证：Debug x64 重新完整编译，0 错误、0 警告。
- 数据库核对：正常 SQLite/WAL 连接可见 1573 条；忽略 `news.db-wal` 的只读方式只能看见主文件中的 1262 条。新增记录仍在 WAL 中，并非另一个数据库。查看或复制运行中的数据库时必须同时处理 `news.db`、`news.db-wal`、`news.db-shm`，或先让程序正常关闭并完成 checkpoint。
