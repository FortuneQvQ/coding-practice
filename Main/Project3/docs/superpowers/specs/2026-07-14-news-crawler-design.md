# 新闻爬虫静态页面提取与安全扩散设计

## 目标

在现有 C++ + libcurl + pugixml + SQLite 架构内，稳定提取静态 HTML 新闻详情并入库；列表页只扩散高置信度新闻子链接，详情页提取成功后不再继续扩散；对内嵌 JSON 和可直接发现的 API 响应提供 curl 方案。无法从 HTTP 响应发现动态数据时，记录原因并停止该分支，避免反复跳转垃圾页面。

## 根因与约束

- 当前未知 URL 会被当成列表页，静态详情页因此被跳过。
- 当前 URL 拼接不支持普通相对路径和规范化，`#`、重复查询参数、重定向结果会造成重复任务。
- 当前链接扩散没有按新闻特征打分，也没有限制单个列表页的子链接数量。
- 当前正文兜底到 `<body>`，会把导航和页脚误判为新闻。
- 当前 JSON 提取依赖固定 `"content"` 文本和脆弱的大括号计数，不能可靠处理转义；curl 本身也不会执行 JS。

约束：不引入浏览器或新的运行时依赖；不保证纯 curl 能执行所有 JS 应用；动态页仅处理 HTTP 响应中已经存在的 JSON、结构化数据或可直接请求的同域 JSON 接口。

## 设计

### 1. 抓取流水线

每个种子建立独立的来源状态：允许的域名、已访问规范 URL、已消耗页面配额、列表页子链接配额。流程为：

1. 规范化种子 URL并入队。
2. curl 获取页面，保留响应 HTML、最终 URL、Content-Type 和 HTTP 状态。
3. 先尝试静态 HTML/结构化数据提取新闻；再尝试页面内 JSON/API 线索。
4. 新闻详情提取成功：校验标题和正文质量，去重后入库，并清空当前页面发现的后续链接。
5. 未提取到新闻：只将同域、非资源、非导航、新闻评分达标的链接按分数排序后入队；每个页面最多扩散固定数量。
6. 达到深度、来源页数或列表子链接配额即停止分支。

### 2. URL 规范化与筛选

新增纯函数：解析 scheme/host/path，正确解析根路径和当前目录相对链接；删除片段，保留必要查询参数；拒绝 `javascript:`, `mailto:`, `tel:`, `data:`、资源后缀和明显退出/登录/搜索/分页导航 URL；限制链接与种子同域。使用规范 URL 作为 visited 和数据库去重键。

新闻链接评分使用 URL 路径、锚文本和查询参数特征：`detail/article/content/info/show`、数字 ID、日期路径、中文标题锚文本加分；`index/list/search/login/download`、纯图片文字和过深资源路径扣分。只把达到阈值的候选入队，并按分数降序、稳定 URL 顺序处理。

### 3. 静态正文提取

标题优先级：`h1`、`article` 内标题、`meta property=og:title`、`title`。正文只在 `article`、ID/class 含 `article/content/main-text/detail` 的容器中寻找 `p`、`div`、`section` 文本；排除 `nav/header/footer/aside/script/style` 和 class/id 含 `nav/menu/footer/sidebar/comment/share` 的节点。计算文本长度、段落数量和中文/字母数字比例，低于质量阈值不入库。正文不再把整个 `<body>` 作为成功兜底。

### 4. JSON/API 方案

对 `<script type="application/ld+json">`、`#__NEXT_DATA__`、常见 `window.__INITIAL_STATE__`/`__NUXT__` 内容使用 nlohmann::json 解析；从 JSON 对象递归寻找 title/date/content/articleBody 字段，并对 content 中的 HTML 再用 pugixml 提取纯文本。对脚本字符串中出现的同域 `.json`、`/api/`、`/ajax/` 地址提取候选，限制为同域、GET 风险低、数量很少，并用 curl 请求 JSON；响应不是 JSON 或未产生高质量新闻时立即停止。

这不能替代 JS 执行：如果真实数据只由运行时计算参数、签名或点击事件产生，curl 不会得到结果。此时输出“未发现可直接访问 JSON/API”，不把脚本、静态资源和导航链接当新闻继续爬。

### 5. 入库与安全边界

数据库打开失败时主程序立即退出，不调用空指针；`addNews` 对无效 News 做最小校验。使用规范 URL、`INSERT OR IGNORE` 保证去重；来源字段从种子来源名或域名补齐。保留现有表结构，不增加迁移需求。

## 测试与验收

- URL：根路径、当前目录相对路径、片段、资源、外域、登录/搜索页和重复 URL。
- 链接筛选：列表页只输出高评分新闻链接，详情页解析成功后输出空后续链接。
- 正文：带导航/页脚的静态详情页只保存正文；纯列表/垃圾页不入库。
- JSON：有效内嵌 JSON 成功；转义大括号和非法 JSON 不导致误解析；无可发现 API 的动态 HTML 不扩散垃圾链接。
- 数据库：重复 URL 不重复写入；数据库打开失败不会继续执行。
- 编译并运行解析测试；若本机缺少 tidy/sqlite/curl 库，给出明确的依赖阻塞，不伪称已通过。
